# Agent Docs

This repository keeps a small agent layer in the repo so a new chat can recover context quickly without re-surveying the whole project.

## Start Here

1. Read `AGENTS.md`.
2. Read `docs/agents/repository-map.md` for the current file inventory.
3. Use `skills/tam-repo-maintainer/SKILL.md` for normal TAM work.
4. Use `skills/chat-handoff/SKILL.md` when the user says `chat handoff` or asks for a context transfer.

## System Flow

1. `scripts/import-tam.ts` reads a workbook or CSV export and writes `data/tam.snapshot.json`.
2. `src/lib/tam/importer.ts` normalizes workbook sheets and extracts embedded workbook images into `public/tam-assets/<dataset>/`.
3. `app/tam/page.tsx` loads the committed snapshot with `src/lib/tam/loadSnapshot.ts`.
4. `app/tam/tam-table.tsx` handles sorting, pagination, inline cell edits, column creation, and row-image controls.
5. `app/api/tam/datasets/[dataset]/sheet/route.ts` and `app/api/tam/datasets/[dataset]/images/route.ts` persist edits back into the snapshot and manual asset folder.

## Agent Artifact Rules

- `docs/agents/repository-map.md` is generated by `npm run agent:refresh`.
- `docs/agents/chat-handoff.md` is regenerated by `npm run handoff`.
- `npm run agent:check` validates both the generated repository map and the repo-local skill metadata.
- Any new high-signal file in `app/`, `src/lib/tam/`, `scripts/`, or `tests/` must get a repository-map summary before `npm test` will pass.
- The command wrappers in `scripts/*.cmd` fail fast with recovery guidance when the synced
  `node_modules` tree is incomplete.

## Commands

- `npm run dev`
- `npm run import:tam -- --input "data/raw/tam.xlsx" --sheet TAM`
- `npm run agent:refresh`
- `npm run agent:check`
- `npm test`
- `npm run handoff`
